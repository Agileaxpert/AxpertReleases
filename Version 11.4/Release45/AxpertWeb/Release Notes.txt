TASK-5076 -- Enhancement: ARMPeriodicNotificationService exe service has been converted to Worker service and avoided hangfire db for job scheduling. Accordingly, default dependency of postgre db has been avoided and Redis functionality introduced for scheduling the Jobs. Incase redis keys cleared / flushall, this worker service needs to be restarted. 
Note: Please note the following points to set this worker service.
      1. The service needs to be started with the following command in command prompt.
         Service Start command ex.:
          sc create ARMPeriodicNotificationWorkerService binPath= "D:\PeriodicNotification\ARMPeriodicNotificationWorkerService.exe D:\PeriodicNotification\appsettings.json"
          sc start ARMPeriodicNotificationWorkerService  
      2. This service wanted to stop or remove from the windows service needs to be run below command.
          Ex.: sc stop ARMPeriodicNotificationWorkerService
               sc delete ARMPeriodicNotificationWorkerService   
      3. Please follow for Linux to start this consumer service. 
          a. Create a folder for your service:
              /home/ubuntu/ARMPeriodicNotificationWorkerService/
          b. Create the systemd service file
              sudo nano /etc/systemd/system/ARMPeriodicNotificationWorkerService.service
          c. Reload system
              sudo systemctl daemon-reload
          d. Enable the service at boot
              sudo systemctl enable ARMPeriodicNotificationWorkerService
          e. Start the service
              sudo systemctl start ARMPeriodicNotificationWorkerService
          f. Check status
              sudo systemctl status ARMPeriodicNotificationWorkerService
          g. To STOP / RESTART the service
              sudo systemctl stop ARMPeriodicNotificationWorkerService
              sudo systemctl restart ARMPeriodicNotificationWorkerService
